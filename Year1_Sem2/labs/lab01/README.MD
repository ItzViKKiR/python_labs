# Лабораторная работа №1 - Класс и инкапсуляция
## Предметная область

Предметная область: рынок недвижимости

Недвижимость включает объекты, которые могут принадлежать владельцам и использоваться для различных целей, например:

* проживания

* сдачи в аренду

* продажи

Каждый объект недвижимости может иметь разные характеристики, такие как цена, ипотека и задолженность по ЖКХ. Также недвижимость может находиться в различных состояниях: свободной или сданной в аренду

## Объект модели

В рамках лабораторной работы реализован  класс Property.

Основная задача объекта — хранить данные о недвижимости и предоставлять методы для управления её состоянием.

### Свойства объекта

Объект недвижимости содержит следующие основные свойства:

* owner — владелец недвижимости

* price — стоимость объекта недвижимости

* rent_term — срок аренды

* utilities — задолженность по ЖКХ

* mortgage — сумма оставшейся ипотеки

* for_rent — можно ли сдавать объект в аренду

* mortgage_active — показывает, есть ли ипотека

* has_debt — показывает, есть ли задолженность по коммунальным платежам

## Состояние объекта

Объект недвижимости имеет логическое состояние:

* rented — объект сдан в аренду

* available — объект доступен для аренды

Состояние изменяется с помощью методов:

* rental_contract() — заключение договора аренды

* close_contract() — закрытие договора аренды

Если объект уже сдан в аренду, повторное заключение договора невозможно.

## Основные методы

Класс содержит бизнес-методы для работы с недвижимостью:

* check_utilities() — проверка задолженности по коммунальным платежам

* check_mortgage() — проверка суммы ипотеки

* owner_info() — получение информации о владельце

Методы для изменения финансового состояния:

* pay_utilities() — оплата коммунальной задолженности

* pay_mortgage() — частичное погашение ипотеки

Методы расчёта стоимости:

* total_rent_price() — общая стоимость аренды

* sell_price() — стоимость продажи недвижимости
